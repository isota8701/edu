# 사용법 — 자동화 교육 콘텐츠 시스템

부모가 매일 저녁 30분~1시간씩 직접 가르치는 홈스쿨링용 자동 콘텐츠 시스템입니다.

> 📁 **두 개의 "progress" 파일이 있어 헷갈릴 수 있습니다.**
>
> | 파일 | 무엇 |
> |------|------|
> | **`/outputs/progress.md`** | 아이 **학습** 진도 (매일 보는 파일) |
> | **`/PROGRESS.md`** | 시스템 **빌드** 진행 (개발자/Claude가 읽는 파일) |
>
> 평소 부모가 열어보는 건 `outputs/progress.md`입니다.

---

## 1. 빠른 시작 (3분)

### 평소 사용 흐름

```
저녁 식사 후
   ↓
Claude Code 실행
   ↓
/today 수학              ← 한 줄로 오늘 자료 생성
   ↓
outputs/daily/YYYY-MM-DD.md 열어서 아이와 학습 (30~40분)
   ↓
/done 수학 — [결과 한 줄 메모]   ← 결과 기록
   ↓
다음 날 같은 흐름
```

### 처음 사용할 때 (한 번만)

```
/plan 수학        # 수학 12주 커리큘럼 생성 (이미 생성됨)
/plan 국어        # 국어 시작 전 한 번
/plan 영어        # 영어 시작 전 한 번
```

> 커리큘럼 없이도 `/today`는 동작하지만, 있으면 단원 진행이 자연스러워집니다.

---

## 2. 5개 명령어 한눈에

| 명령 | 언제 쓰는지 | 인수 |
|------|------------|------|
| **`/today`** | 오늘 학습 자료가 필요할 때 (매일 사용) | `[과목]` `[주제\|자유 메모]` |
| **`/done`** | 학습 끝나고 결과 알려줄 때 (매일 사용) | `[과목] — [결과 메모]` |
| **`/plan`** | 과목 커리큘럼을 짜거나 다시 짤 때 | `[과목]` |
| **`/level`** | 단원 평가나 위치 확인이 필요할 때 | `[과목]` `[주제]` |
| **`/review`** | 약한 부분 복습이 필요할 때 | `[주제]` |

---

## 3. 매일 쓰는 두 명령어

### 3.1 `/today` — 오늘 자료 생성

가장 많이 쓰는 명령. 과목 하나만 적으면 시스템이 알아서 판단해서 자료 생성.

```bash
# 가장 일반적인 사용
/today 수학
/today 국어
/today 영어

# 주제 명시 (커리큘럼 무시하고 그 주제로)
/today 국어 시사
/today 국어 비문학
/today 국어 문학

# 자유 컨텍스트 (시간·컨디션 등)
/today 수학 "오늘 시간 20분만"
/today "오늘 피곤해 30분만"     # 과목도 자동 선택
/today                         # 인수 없음 → 그날 요일 메인 과목 자동 선택

# enrichment (간헐, 명시 호출 시만)
/today 과학
/today 인문학
```

**자동으로 일어나는 일:**
- 학생 프로필·진도·커리큘럼·직전 세션 오답 모두 검토
- 요일에 맞는 메인 과목 추천
- 새 단원이면 가벼운 진단 + 개념 도입 자동 추가
- 단원 종료 시점이면 평가 5문제 자동 추가
- 누적 오답 패턴이 있으면 그 유형 1~2문제 재출제

**결과**: `outputs/daily/YYYY-MM-DD.md` 파일이 생성됨. 이 파일을 열어 아이와 학습.

### 3.2 `/done` — 결과 기록

학습 끝나고 무조건 한 줄. 메모는 자유롭게.

```bash
# 정석
/done 수학 — 본문제 5개 중 4개 정답, (3)번 헷갈림. 4단 워밍업 △ 1개

# 더 짧게도 OK
/done 수학 — 다 맞음, 잘함
/done 영어 — ai 발음 잘함, 쓰기는 a/d 혼동

# 메모 없이 호출하면 시스템이 물어봄
/done
```

**자동으로 일어나는 일:**
- `outputs/progress.md`에 세션 로그·오답 추가
- `student/profile.md`의 취약점 갱신
- 다음 세션 예고 1줄 출력 (어떤 과목, 어떤 보강 예정)

> 결과가 안 좋은 날도 부담 없이 적기. "다음에 보강하면 OK"가 시스템의 기본 입장.

## 4. 가끔 쓰는 세 명령어

### 4.1 `/plan` — 커리큘럼 만들기/다시 짜기

처음 그 과목 시작할 때 한 번 + 단원 종료 후 큰 변화 있을 때.

```bash
/plan 수학          # 수학 12주 단원 시퀀스 생성
/plan 국어          # 국어 지문 주제 시퀀스 생성
/plan 영어          # 파닉스 + 단어 트랙 생성
/plan enrichment    # 과학·인문학 주제 풀 카탈로그 생성
/plan               # 어떤 과목 짤지 묻고 진행
```

**결과**: `outputs/curriculum/{과목}.md` 생성/갱신.

### 4.2 `/level` — 진단·평가

두 가지 상황에서:
- **단원 시작 시**: 가벼운 위치 확인 (1~2문제)
- **단원 종료 시**: 5문제 평가

```bash
/level 수학              # 현재 단원에 맞춰 자동 판단
/level 영어 파닉스        # 영역 명시
/level 수학 종료         # 단원 종료 평가 명시
/level 수학 시작         # 단원 시작 진단 명시
```

> 사실 `/today`가 단원 경계를 자동 감지해서 진단·평가를 끼워 넣어주기 때문에 `/level`을 따로 부를 일은 자주 없음. 특정 영역만 따로 진단하고 싶을 때 사용.

### 4.3 `/review` — 약한 부분 복습

오답이 누적된 유형, 또는 부모가 보기에 보강 필요한 영역.

```bash
/review                # 직전 세션 오답 자동 추출
/review 받아내림        # 특정 유형
/review 구구단 4단      # 더 구체적
/review 수학           # 수학 전체에서 약한 영역 자동 선정
```

**결과**: 같은 유형 변형 문제 4~6개 + 단계별 풀이 1~2개. 같은 날 daily 파일에 "복습" 섹션으로 추가.

## 5. 요일 로테이션 (디폴트)

| 요일 | 메인 과목 |
|------|----------|
| 월 | 수학 + 영어 |
| 화 | 국어 + 영어 |
| 수 | 수학 + 국어 |
| 목 | 영어 + 수학 |
| 금 | 국어 + 영어 |
| 토 | 수학 + 국어 |
| 일 | 자유 (약했던 영역 가벼운 복습 권장) |

주당 빈도: 수학 4회 · 국어 4회 · 영어 4회.

**enrichment(과학·인문학)는 자동 배치 X** — `/today 과학` 또는 `/today 인문학`으로 명시 호출할 때만.

> 로테이션 변경하고 싶으면 `CLAUDE.md` 4장을 직접 수정하거나, "월요일은 수학 대신 국어로 바꿔줘"라고 Claude에게 말하면 됨.

## 6. 상황별 가이드

### "오늘 시간이 없어"
```
/today "오늘 시간 20분"
```
시스템이 분량을 줄여서 자료 생성. 빠르게 끝낼 수 있는 핵심만.

### "오늘 아이가 틀렸던 부분 다시 하고 싶어"
```
/review 받아내림
```
같은 유형 문제 변형해서 새로 만들어줌.

### "이 단원 다 끝낸 것 같은데 평가하고 싶어"
```
/level 수학 종료
```
5문제 평가지 생성. 4문제 이상 맞으면 다음 단원으로 넘어감.

### "주말이라 가볍게 다른 거 하고 싶어"
```
/today 과학
/today 인문학
```
간헐 코스. 부담 없는 흥미 위주 활동.

### "아이가 학교에서 새 단원 시작했어"
```
/plan 수학
```
이미 있는 커리큘럼을 학교 진도에 맞춰 다시 짜줌. 변경 이력은 파일 하단 "조정 기록"에 남음.

## 7. 파일 구조

```
project/
├── CLAUDE.md, USAGE.md, PROGRESS.md, spec.md   # 시스템 문서
├── data/academy_research.md                    # 학원 패턴 참고 베이스
├── student/profile.md                          # 아이 수준·취약점 (자동 갱신)
├── outputs/
│   ├── progress.md                             # 학습 진도 로그 (자동 갱신)
│   ├── curriculum/{math,korean,english,enrichment}.md
│   └── daily/YYYY-MM-DD.md                     # 그날의 학습 자료
└── .claude/                                    # 시스템 내부 (수정 불필요)
    ├── agents/
    └── commands/
```

**부모가 자주 열어보는 파일:**
- `outputs/daily/YYYY-MM-DD.md` — 매일 학습할 자료
- `outputs/progress.md` — 진도 확인할 때
- `student/profile.md` — 아이 현재 수준 확인할 때
- `outputs/curriculum/{과목}.md` — 다음에 뭐 배울지 확인할 때

**부모가 잘 열지 않는 파일:**
- `.claude/` 하위 — 시스템 내부 동작 정의. 수정하지 않음.
- `data/academy_research.md` — 분기 1회 갱신 정도. 평소엔 그대로.
- `CLAUDE.md` — 시스템 운영 규칙. 큰 방향 바꿀 때만 수정.

## 8. 자주 묻는 질문

**Q. 명령어 이름을 외워야 하나?**
A. `/today`와 `/done` 두 개만 알면 매일 사용 가능. 나머지는 필요할 때 이 문서 참고.

**Q. 일요일은 뭘 해야 하나?**
A. 디폴트는 "자유". `/today` 인수 없이 부르면 시스템이 그 주에 약했던 영역으로 가벼운 복습을 추천. 또는 `/today 과학` 같은 enrichment로 쉬어가기.

**Q. 학원 자료를 그대로 쓰는 건가?**
A. 아닙니다. `data/academy_research.md`에 학원 커리큘럼 *패턴*만 추상화되어 있고, 실제 문제는 매번 새로 생성됩니다. 표절 X.

**Q. 결과 메모를 적기 귀찮은 날은?**
A. `/done` 만 써도 됨. 시스템이 1~2개 질문으로 물어봄. 답하기 어려우면 "패스"라고 해도 OK.

**Q. 아이가 단원을 통과 못 했을 때?**
A. 자동으로 `/review` 1주 보강이 권장됨. 강제는 아님 — 부모가 판단.

**Q. 시스템이 계획한 진도가 너무 빠르거나 느릴 때?**
A. `/plan {과목}`으로 다시 짜거나, Claude에게 "단원 4 기간을 3주로 늘려줘" 같이 자연어로 요청해도 됨.

**Q. `outputs/progress.md`나 `student/profile.md`를 직접 수정해도 되나?**
A. 됨. 다만 형식이 깨지지 않게 주의. 다음 `/done`에서 시스템이 갱신할 때 충돌하지 않도록 큰 구조는 유지.
