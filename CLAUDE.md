# CLAUDE.md — 프로젝트 운영 가이드

## 0. 한 줄 요약

초등학교 2학년 자녀를 홈스쿨링하는 부모를 위한 **맞춤형 교육 콘텐츠 자동 생성 시스템**.
강남·잠실·서초·목동·분당·노원 학원 커리큘럼 패턴을 참고하여 **사고력 중심**의 일일 학습 자료를 매일 저녁 자동 생성한다.

목적: 기초 정착 + 한 단계 위 사고력 + 자신감.

---

## 1. 세션 시작 시 반드시 할 일 (Claude 내부 루틴)

새 대화가 열리면 **순서대로** 읽고 상태를 파악한 뒤, 사용자(부모)에게 다음 행동을 제안한다.

1. **`PROGRESS.md`** — 시스템 빌드 진행 상황. 항상 먼저.
2. **`outputs/progress.md`** — 아이의 *학습* 진도 (있으면).
3. **`student/profile.md`** — 아이의 현재 수준·취약 포인트.
4. **`outputs/curriculum/{과목}.md`** — 해당 과목의 다주차 계획 (있으면).
5. **`data/academy_research.md`** — 학원 커리큘럼 참고 베이스 (있으면).

그 다음 사용자에게:
- 시스템이 미완성이면 → `PROGRESS.md` 섹션 5 다음 할 일 중 하나 제안
- 시스템이 준비됐으면 → 요일에 맞는 `/today` 실행 제안

### ⚠️ 두 개의 "progress" 파일 구분 (헷갈리지 말 것)

| 파일 경로 | 추적 대상 | 갱신 주체·시점 |
|-----------|----------|----------------|
| **`/PROGRESS.md`** (루트) | 시스템 빌드·설계 결정(D1~D21)·아키텍처 | 시스템 변경 시 Claude 수동 갱신 |
| **`/outputs/progress.md`** | 아이 학습 진도·세션 로그·오답·누적 복습 트래커 | `/done` 호출 시 자동 갱신 |

**규칙**:
- "progress.md"라고만 보이면 → **반드시 경로 확인**. 루트면 시스템, `outputs/`면 학습.
- 새 코드·문서에서는 가능하면 전체 경로로 표기 (`/PROGRESS.md` vs `outputs/progress.md`).

## 3. 5개 명령어 (부모 진입점)

| 명령 | 역할 | 인수 |
|------|------|------|
| `/today` | 오늘 세션 자료 생성 → `outputs/daily/YYYY-MM-DD.md` | `[과목] [주제\|자유컨텍스트]` (모두 선택) |
| `/done` | 결과 기록 + `outputs/progress.md`·`student/profile.md` 갱신 | `[메모]` |
| `/plan` | 과목 커리큘럼 생성/갱신 → `outputs/curriculum/{과목}.md` | `[과목]` |
| `/level` | 가벼운 진단 (단원 시작/종료) | `[과목]` |
| `/review` | 취약점 복습 문제 재생성 | `[주제]` |

### `/today`의 내부 라우팅 로직 (핵심)
```
1. 인수 파싱:
   - 과목 명시(수학/국어/영어/enrichment) → 그 과목으로
   - 인수 없거나 자유 텍스트 → 요일 로테이션 + 자유 컨텍스트로 과목 결정
2. 학생 프로필·진도·해당 과목 커리큘럼·학원 리서치 베이스 읽기
3. 직전 세션 결과를 보고 복습 비중·오답 재출제 판단
4. 새 단원 진입이면 개념 도입 + 가벼운 진단 1~2문제 자동 추가
5. 단원 종료 시점이면 5문제 평가 추가 (= /level 자동 호출)
6. 해당 subagent에 모든 컨텍스트 전달 → 콘텐츠 생성
7. outputs/daily/YYYY-MM-DD.md 작성
```

## 4. 요일 로테이션 디폴트

매일 저녁 메인 2과목씩 (30분~1시간 안에 끝낼 수 있는 분량):

| 요일 | 메인 |
|------|------|
| 월 | 수학 + 영어 |
| 화 | 국어 + 영어 |
| 수 | 수학 + 국어 |
| 목 | 영어 + 수학 |
| 금 | 국어 + 영어 |
| 토 | 수학 + 국어 |
| 일 | 가벼운 복습 또는 부모가 명시 호출 (enrichment 포함) |

주당 빈도: 수학 4회 · 국어 4회 · 영어 4회.
**enrichment(과학·인문학)는 자동 배치 X** — 부모가 `/today enrichment` 또는 `/today 과학` 등으로 명시 호출할 때만.

> 부모가 `/today` (인수 없이) 호출하면 그 요일의 메인 2개 중 하나를 추천하거나 둘 다 묶어서 출력. 자유 텍스트(예: "오늘 시간 20분")가 있으면 그에 맞춰 분량·과목 조정.

## 6. 콘텐츠 생성 원칙

- **사고력 중심**: 단순 반복·연산 드릴 X. 이유 묻기, 여러 풀이, 패턴 발견, 추론 문제 우선.
- **학교 진도 sync**: 학교에서 다루는 단원과 시기 맞춰 보강.
- **학원 커리큘럼 참고**: 표절 X, 레벨감·구조·문제 유형만 참고.
- **하루 분량**: 30분~1시간 안에 부담 없이 끝낼 수 있는 양.
- **긍정적 프레이밍**: 오답 지도 스크립트는 "어디서 멈췄는지 같이 보자" 톤.

### 출력 형식 (과목별)
- **수학**: 부모용 지도지. 워밍업 → 개념 확인 → 본문제 → 사고력 도전 → 정답·풀이.
- **국어**: 아이가 직접 읽을 지문 → 사실확인·추론·의견 문제 → 토론 질문 → 부모용 해설.
- **영어**: 패턴(파닉스) 소개 → 단어 카드 → 듣기/따라 읽기 → 쓰기 → 짧은 문장. 부모 발음 노트 포함.
- **enrichment**: 짧은 흥미 자료 → 관찰·토론 질문 → 자유 활동.

**금지 섹션**: "부모 컨디션 체크", "화 날 때 루틴" 같은 마인드 코칭 섹션은 출력에 넣지 않는다.

## 7. 아이 현재 수준 스냅샷 (2026-04-26)

> 실시간 반영은 `student/profile.md`. 여기는 참고용.

- **수학**: 학교 진도 — 세 자리 수, 곱셈의 이해, 두 자리 덧뺄셈, 도형. 받아올림/내림은 익숙하지는 않지만 풀 수 있음. 구구단은 4단부터 랜덤 응답이 아직 느림.
- **국어**: 학교 진도 미파악. 받아쓰기는 부모가 직접 진행(시스템 콘텐츠 생성 X). 시스템에서 다룰 영역은 **신문 지문 + 토론**.
- **영어**: CVC 가능, 이중모음 불안정, is/are·have/has 부정확, 소문자 학습 단계, input 부족. 부모 발음 지도 가능(영어권 거주 경험).

## 8. 작업 규칙

- `spec.md`는 참고용, 수정하지 않는다.
- 새 결정은 `PROGRESS.md` 섹션 2에 행 추가 (이력 유지).
- `PROGRESS.md` 섹션 4 체크박스는 즉시 갱신.
- 사용자가 학습 결과를 리포트백하면 `outputs/progress.md`·`student/profile.md` 갱신.
- 이 파일(`CLAUDE.md`)은 철학·구조 변경 시에만 수정. 일상은 `PROGRESS.md`에서.
