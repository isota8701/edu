과목 또는 단원에 대한 가벼운 진단 평가를 생성한다.

## 사용법
```
/level 수학                # 현재 단원에 맞춘 진단
/level 수학 곱셈           # 특정 주제 진단
/level 영어 파닉스          # 영어 파닉스 진단
/level                    # 어떤 과목·주제를 진단할지 대화로 물어봄
```

## 두 가지 모드 (D3 — 둘 다 사용)

### 모드 A: 단원 시작 진단 (가벼움)
- **2~3문제** — 위치 확인용
- 결과는 다음 세션 콘텐츠 난이도 조정에만 사용
- 부담 없이 풀 수 있는 톤

### 모드 B: 단원 종료 평가
- **5문제** — 단원 학습이 정착됐는지 확인
- 난이도: 쉬움 1 / 표준 3 / 응용 1
- 결과는 `outputs/progress.md`에 기록 + 다음 단원 진입 결정

자동 판단:
- 인수에 "시작" / "도입" 류 키워드 → 모드 A
- 인수에 "평가" / "종료" / "마무리" 류 → 모드 B
- 키워드 없으면 `outputs/progress.md` 보고 단원 위치로 판단

## 실행 루틴
1. `student/profile.md`, `outputs/progress.md`, 해당 과목 커리큘럼 읽기
2. 해당 과목 subagent에게 진단 형식 요청
3. 결과 출력은 `outputs/daily/YYYY-MM-DD-{과목}.md`에 섹션으로 추가하거나 별도 `outputs/level/{date}_{subject}.md`로 저장 (디폴트: 과목별 daily에 추가)
4. 결과 입력은 `/done`으로 받음

## 출력 형식 (모드 B 예시)
```markdown
## 수학 단원 평가 — [단원명]
1. (쉬움) ...
2. (표준) ...
3. (표준) ...
4. (표준) ...
5. (응용) ...

### 부모용 정답
...

### 통과 기준
- 5개 중 4개 이상 → 다음 단원 진입
- 3개 이하 → 1주 보강 후 재평가
```
