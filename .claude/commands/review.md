취약 포인트나 특정 주제에 대한 복습 문제를 재생성한다.

## 사용법
```
/review 받아내림              # 누적 오답이 있는 특정 유형 복습
/review 구구단 4단             # 더 구체적 주제
/review 수학                  # 과목 전체에서 약한 영역 자동 선정
/review                      # 직전 세션 오답 패턴 자동 추출
```

## 실행 루틴

1. **읽기**:
   - `student/profile.md` — 취약 포인트
   - `outputs/progress.md` — 누적 오답 기록
   - 직전 1~3회 `outputs/daily/*.md` — 최근 오답
2. **주제 식별**:
   - 인수가 명시적이면 그대로
   - 과목만 명시 → 누적 오답 빈도 1위 유형
   - 인수 없음 → 직전 세션 오답 유형
3. **해당 과목 subagent 호출** (review 모드):
   - 같은 유형 변형 문제 4~6개
   - 난이도는 직전보다 살짝 낮게 시작 → 마지막 1~2개는 같은 수준
   - 풀이 과정을 단계별로 노출하는 "함께 풀어보기" 1~2개 포함
4. **출력**: `outputs/daily/YYYY-MM-DD.md`에 "복습 — [주제]" 섹션으로 추가
5. **마무리 메시지**:
   ```
   복습 자료 생성: [주제]. 끝나면 /done 으로 결과 알려주세요.
   /done 에 "복습 [주제] — N개 중 M개" 형식으로 적어주시면 진도에 자동 반영됩니다.
   ```

## 출력 형식 (수학 예시)
```markdown
## 복습 — 받아내림 뺄셈

### 함께 풀어보기 (1~2개)
[단계별 풀이 노출. 부모가 같이 따라가는 형식]

### 복습 문제
1. ...
2. ...
3. ...
4. ...

### 부모용 정답·핵심 포인트
...

### 통과 기준
- 4개 중 3개 이상 → 취약 포인트 해제 후보
- 2개 이하 → 다음 세션에 1~2문제 더 출제
```

## 호출 시 주의
- 복습은 부담 줄이는 분위기로. "다시 같이 보자" 톤.
- 같은 유형이라도 표현·문맥을 살짝 바꿔 새 문제처럼 느끼게.
