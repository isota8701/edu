오늘 세션 자료를 생성한다. 스마트 라우터 — 인수가 적어도 시스템이 알아서 판단한다.

## 사용법
```
/today                              # 인수 없음: 요일 로테이션에 따라 그날 메인 과목 자동 선택
/today 수학                          # 과목 명시
/today 국어 시사                     # 과목 + 주제
/today 수학 "오늘 시간 20분"          # 과목 + 자유 컨텍스트
/today "오늘 피곤해 30분만"           # 자유 컨텍스트만 (과목도 자동 판단)
/today 과학                          # enrichment 명시 호출 (과학)
/today 인문학                        # enrichment 명시 호출 (인문학)
```

## 실행 루틴

1. **읽기 (순서대로)**:
   - `student/profile.md`
   - `outputs/progress.md`
   - `data/academy_research.md`
   - 해당 과목의 `outputs/curriculum/{과목}.md` (존재하면)
   - 직전 1~2회 `outputs/daily/*.md` (오답 재출제 판단)

2. **인수 파싱**:
   - 과목 키워드 감지: `수학` `국어` `영어` `과학` `인문학` `enrichment`
   - 과목이 없으면 → 오늘 요일의 메인 2개 중 하나 선택
     - 직전 세션이 둘 중 하나였으면 다른 쪽 우선
     - 자유 컨텍스트에 시간 제약 있으면 짧게 끝낼 과목 우선
   - 자유 컨텍스트 (큰따옴표 안) → 분량·난이도 조정 신호

3. **요일 로테이션 (CLAUDE.md 4장)**:
   | 요일 | 메인 |
   |------|------|
   | 월 | 수학 + 영어 |
   | 화 | 국어 + 영어 |
   | 수 | 수학 + 국어 |
   | 목 | 영어 + 수학 |
   | 금 | 국어 + 영어 |
   | 토 | 수학 + 국어 |
   | 일 | (자유 — 약했던 영역 가벼운 복습 추천) |

4. **상태 판단** (자동):
   - 새 단원 진입? → 가벼운 진단 1~2문제 + 개념 도입
   - 단원 종료 시점? → `/level`처럼 평가 5문제 추가
   - 직전 세션 오답 패턴 있음? → 해당 유형 1~2문제 재출제

5. **Subagent 호출**:
   - 수학 → math-tutor
   - 국어 → korean-tutor
   - 영어 → english-tutor
   - 과학 / 인문학 / enrichment → enrichment-tutor (sub-topic 전달)

6. **출력 작성**: `outputs/daily/YYYY-MM-DD-{과목}.md`
   - 과목마다 **별도 파일**. 같은 날 여러 과목을 진행해도 파일을 합치지 않는다.
   - 파일명 규칙: `2026-05-01-수학.md` / `2026-05-01-국어.md` / `2026-05-01-영어.md`
   - enrichment는 sub-topic으로: `2026-05-01-과학.md`

7. **마무리 메시지**:
   ```
   파일 생성 완료: outputs/daily/YYYY-MM-DD-{과목}.md
   세션 끝나면 /done [메모] 로 결과 알려주세요.
   ```

## 출력 파일 헤더
모든 daily 파일은 다음 헤더로 시작 (날짜·요일 + 과목 표기 필수):
```markdown
# YYYY-MM-DD (요일) — {과목}

세션 끝나면 `/done [결과 메모]` 로 진도 갱신.

---
```

- `# 2026-05-01 (금) — 영어` → 파일명 `2026-05-01-영어.md`
- enrichment: `# 2026-05-01 (금) — 과학` → 파일명 `2026-05-01-과학.md`

이 헤더 아래에 subagent가 생성한 과목별 콘텐츠가 들어간다.

## 금지 사항
- "부모 컨디션 체크", "화 날 때 루틴" 같은 마인드 코칭 섹션 **절대 금지** (사용자 명시 요청 — 2026-04-26)
