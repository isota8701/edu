오늘 학습 결과를 기록하고 진도 파일을 갱신한다.

## 사용법
```
/done                                # 메모 없이 호출하면 대화로 결과를 물어봄
/done 수학 — 8문제 중 6개 맞음, (4)번 받아내림 틀림, 구구단 4×7 헷갈려함
/done 국어 — 지문 잘 읽음, 추론 문제 1번 어려워함
/done 영어 — ai 발음 안정적, train·rain 잘 읽음. 쓰기에서 a/d 혼동
/done 2026-05-10 영어 완료. cat은 읽었고 map은 막힘. b/d를 헷갈려함
```

Codex에서 slash command가 모델 도달 전에 막히면 다음 입력도 `/done`으로 처리한다.
```
done 2026-05-10 영어 완료. cat은 읽었고 map은 막힘
'/done' 2026-05-10 영어 완료. cat은 읽었고 map은 막힘
오늘 학습 결과: 2026-05-10 영어 완료. cat은 읽었고 map은 막힘
```

## 실행 루틴

1. **읽기**: `outputs/progress.md`, `student/profile.md`, `outputs/learning-log.md`(없으면 생성), 오늘 또는 최근의 `outputs/daily/YYYY-MM-DD-{과목}.md`
2. **메모 분석**:
   - 날짜: `YYYY-MM-DD`, `YYYY.MM.DD`, `M/D` 형식을 인식. 없으면 오늘 날짜 또는 가장 최근 daily 파일 기준.
   - 과목: `수학` `국어` `영어` `과학` `인문학` 감지. 없으면 daily 파일과 메모 내용으로 추론.
   - 맞은 문제 수 / 전체 문제 수
   - 틀린 문제 유형 (오답 패턴)
   - 아이 컨디션·반응 (언급 있으면)
   - 새 단원 첫 반응 (구구단·파닉스 등)
3. **`outputs/progress.md` 갱신**:
   - 세션 로그 테이블에 오늘 행 추가
   - 오답 기록 테이블에 오답 추가
   - 현재 단원 상태 갱신 (진행 중 / 단원 종료 임박 등)
4. **`student/profile.md` 갱신**:
   - 취약 포인트에 오늘 확인된 패턴 추가/갱신
   - 안정화된 영역은 취약 → 안정으로 이동
5. **daily 파일 완료 표시**:
   - 대상 파일: `outputs/daily/YYYY-MM-DD-{과목}.md`
   - `/done` 반영 후 `outputs/daily/YYYY-MM-DD-{과목}-done.md`로 이름을 바꾼다.
   - 이미 `-done.md`이면 이름은 유지한다.
   - 원본 파일이 없고 `-done.md`만 있으면 그 파일을 대상 daily 파일로 본다.
   - 같은 날짜·과목 파일이 여러 개라 추론이 불안하면 변경 전에 한 번만 확인한다.
6. **`outputs/learning-log.md` 갱신**:
   - 없으면 새로 만든다.
   - 최신 항목이 위로 오게 추가한다.
   - git log와 비슷하게 한 세션을 6~8줄로 짧게 정리한다.
   - 권장 형식:
     ```markdown
     commit YYYY-MM-DD-{과목}
     Date: YYYY-MM-DD ({요일})
     Subject: {과목}
     Topic: {오늘 다룬 핵심 1줄}
     Result: {부모 피드백 요약 1줄}
     Next: {다음 보강 포인트 1줄}
     File: outputs/daily/YYYY-MM-DD-{과목}-done.md
     ```
   - 중간 코멘트가 필요하면 `Comment:` 줄을 1줄만 추가한다.
7. **다음 세션 예고 1줄 출력**:
   ```
   기록 완료. outputs/daily/YYYY-MM-DD-{과목}-done.md로 완료 표시했고, 다음 세션은 [보강 포인트]를 반영합니다.
   ```

## 호출 시 주의
- 메모가 모호하거나 누락되면 1~2개 명확화 질문을 던지고 답을 받아 갱신.
- 과목이 명시되지 않았으면 최근 daily 파일과 메모 내용을 대조해 과목을 추론한다. 추론이 불안하면 한 번만 확인한다.
- 날짜가 명시되어 있으면 오늘 날짜보다 우선한다. 예: `/done 2026.05.10 영어 완료 ...`는 2026-05-10 영어 daily 파일을 찾는다.
- daily 파일명 완료 표시와 `outputs/learning-log.md` 갱신은 `/done`의 필수 작업이다.
- 갱신은 한 번에 끝내되, 학습 기록 왜곡 위험이 있으면 저장 전에 짧게 확인한다.
- 학습 결과가 좋지 않은 날에도 부정적 표현 사용 금지 — "다음에 보강하면 OK".
